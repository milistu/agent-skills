# Designing for iPadOS

Guidelines for designing apps and games that feel at home on iPad.

## Device Characteristics

- **Display**: Large, high-resolution display
- **Ergonomics**: People hold iPad, set it on surfaces, or use stands. Typical viewing distance is within ~3 feet
- **Inputs**: Multi-Touch gestures, virtual keyboards, attached keyboard/trackpad, Apple Pencil, voice — users often combine multiple input modes
- **App interactions**: Range from quick actions to hours-long sessions. Users frequently have multiple apps open simultaneously, view more than one app onscreen at once, and use inter-app capabilities like drag and drop

## System Features

- Multitasking
- Widgets
- Drag and drop

## Best Practices

- **Maximize the large display**: Elevate content people care about. Minimize modal interfaces and full-screen transitions. Position onscreen controls where they're easy to reach but not in the way.
- **Adapt content density to context**: Use viewing distance and input mode to determine the size and density of onscreen content.
- **Support multiple input modes**: Let people use Multi-Touch gestures, physical keyboard/trackpad, or Apple Pencil. Consider supporting unique interactions that combine multiple input modes.
- **Adapt to appearance changes seamlessly**: Handle device orientation, multitasking modes, Dark Mode, Dynamic Type, and transition to macOS (Mac Catalyst). Let people choose configurations that work best for them.
