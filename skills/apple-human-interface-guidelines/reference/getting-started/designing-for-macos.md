# Designing for macOS

Guidelines for designing apps and games that feel at home on macOS, covering device characteristics, interaction patterns, and best practices.

## Device Characteristics

- **Display:** Large, high-resolution display; users can extend workspace with additional displays (including iPad)
- **Ergonomics:** Stationary use, typically on a desk/table. Viewing distance ~1–3 feet
- **Inputs:** Physical keyboards, pointing devices, game controllers, Siri — often used in combination
- **App interactions:** Sessions range from quick tasks (minutes) to deep concentration (hours). Multiple apps open simultaneously; expect smooth active/inactive transitions

## Key System Features

- The menu bar
- File management
- Full-screen mode
- Dock menus

## Best Practices

- **Leverage large displays** — Present more content in fewer nested levels with less modality. Maintain comfortable information density; don't make people strain to view content.
- **Flexible window management** — Let people resize, hide, show, and move windows to fit their work style and device configuration. Support full-screen mode for distraction-free contexts.
- **Menu bar** — Use it to give people easy access to all commands they need.
- **High-precision input** — Help people take advantage of precise input modes for pixel-perfect selections and edits.
- **Keyboard shortcuts** — Handle shortcuts to accelerate actions and support keyboard-only work styles.
- **Personalization** — Let people customize toolbars, configure windows to display preferred views, and choose colors and fonts for the interface.
