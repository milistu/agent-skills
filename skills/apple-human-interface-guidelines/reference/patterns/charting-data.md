# Charting Data

Guidelines for presenting data in charts to communicate information with clarity and appeal across Apple platforms.

## When to Use Charts

Charts help people perform data-driven tasks such as:
- Analyzing trends based on historical or predicted values
- Visualizing current state of a process, system, or quantity that changes over time
- Evaluating different items by comparing data across multiple categories

If you don't need to convey information *about* data or help people analyze it, consider offering data in a list or table instead.

## Best Practices

- **Use a chart to highlight important information about a dataset.** Charts are visually prominent — clearly communicate what people can learn from the data they care about.
- **Keep charts simple; let people choose when they want additional details.** Too much data makes a chart visually overwhelming and obscures relationships. Consider letting people reveal detail gradually (e.g., different levels of detail or subsets of data). Offer progressively more functional versions to teach interactivity.
- **Make every chart accessible.** Provide accessibility labels describing chart values and components, plus accessibility elements for interaction. See Charts > Enhancing the accessibility of a chart.

## Designing Effective Charts

- **Prefer common chart types** (bar charts, line charts). People already know how to read them.
- **For novel chart types, help people learn to interpret them.** Example: Activity introduces rings by animating them individually, mapping each ring to move, exercise, and stand metrics.
- **Examine data from multiple levels to find enhancing details:**
  - **Macro level:** High-level summaries (totals, averages)
  - **Mid level:** Useful subsets of data
  - **Individual data points:** Specific values or items worth highlighting
- **Add descriptive text** — titles, subtitles, annotations, and summary headlines help people grasp essential information at a glance. Example: Weather shows "Chance of light rain in the next hour" above hourly forecasts. Descriptive text doesn't replace accessibility labels.
- **Match chart size to functionality, topic, and detail level.** Charts must be large enough to display details comfortably and support interactivity. Small charts work for glanceable information or previews of larger versions.
- **Prefer consistency across multiple charts** that serve similar purposes — same type, style, and visual approach so people can transfer learning between charts. Use different types/styles only to highlight meaningful differences.
- **Maintain continuity among charts using the same data.** Use consistent chart type, colors, annotations, layouts, and descriptive text to signal the dataset is the same across different views or zoom levels.

## Platform Considerations

No additional platform-specific considerations for iOS, iPadOS, macOS, tvOS, visionOS, or watchOS.
